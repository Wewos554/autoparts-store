# AutoParts Store - Интернет-магазин автозапчастей

## 📋 О проекте

**Название приложения:** AutoParts Store

**Краткое описание:**  
AutoParts Store — это современный интернет-магазин автозапчастей, позволяющий пользователям просматривать каталог, искать товары по бренду и категории, добавлять понравившиеся запчасти в избранное, а также управлять собственными объявлениями. Приложение разработано с учетом потребностей автовладельцев и автомехаников.

**Целевая аудитория:**  
- Автовладельцы, желающие самостоятельно подобрать запчасти
- Автомеханики и автосервисы, нуждающиеся в быстром поиске деталей
- Продавцы автозапчастей, желающие размещать свои товары

**Ключевой функционал:**
- 🔐 Регистрация, аутентификация и управление профилем
- 📦 Просмотр каталога автозапчастей
- 🔍 Поиск и фильтрация по бренду, категории и цене
- 📊 Сортировка по дате и цене
- ❤️ Добавление товаров в избранное
- ➕ Добавление и управление собственными товарами
- 🛒 Корзина и оформление заказов
- 📱 Адаптивный дизайн для мобильных устройств

## 🛠️ Технологический стек

### Frontend
| Компонент | Выбор | Обоснование |
|-----------|-------|-------------|
| Язык | **Vanilla JavaScript (ES6+)** | Простота, отсутствие необходимости в тяжелых фреймворках, нативный код работает быстро |
| Роутинг | **Нативный (HTML5 History API + серверная маршрутизация)** | Express на бэкенде отдает HTML страницы, что упрощает архитектуру |
| State-менеджмент | **localStorage + глобальные переменные** | Для хранения токена авторизации и имени пользователя достаточно простого решения |
| Стили | **CSS3 (Flexbox, Grid, Media Queries)** | Адаптивный дизайн без дополнительных зависимостей |
| HTTP клиент | **Fetch API** | Встроенный в браузер, современный, Promise-based |

### Backend
| Компонент | Выбор | Обоснование |
|-----------|-------|-------------|
| Язык | **Node.js** | JavaScript на сервере — единый язык с фронтендом, огромное сообщество |
| Фреймворк | **Express.js** | Минималистичный, быстрый, де-факто стандарт для Node.js REST API |
| Аутентификация | **JWT (JSON Web Tokens)** | Стандарт для stateless аутентификации, не требует хранения сессий |
| Хеширование паролей | **bcrypt** | Надежное хеширование паролей с солью |

### База данных
| Компонент | Выбор | Обоснование |
|-----------|-------|-------------|
| СУБД | **MongoDB** | Гибкая схема (подходит для часто меняющейся структуры товаров), легкая интеграция с Node.js |
| ODM | **Mongoose** | Удобное моделирование данных, валидация, middleware |

### Инфраструктура
| Компонент | Сервис | Обоснование |
|-----------|--------|-------------|
| Хостинг фронтенда | **Netlify** | Бесплатный, простой деплой, автоматический HTTPS |
| Хостинг бэкенда | **Railway** / **Render** | Бесплатные кредиты, простая настройка, автоматический деплой из GitHub |
| База данных | **MongoDB Atlas** | Бесплатный кластер M0, облачное решение с автоматическим бэкапом |
| Контроль версий | **Git + GitHub** | Популярный сервис, CI/CD интеграция с хостингами |

## 🎨 Визуальное проектирование интерфейса (Вариант В: текстовое описание)

### Страница 1: Главная (`/`)
**Тип доступа:** Публичный

**Основные блоки:**
1. **Шапка (Header)** — логотип, навигация (Главная, О нас, Контакты, Войти/Регистрация или Профиль/Избранное/Выйти)
2. **Заголовок** — "Каталог автозапчастей"
3. **Блок фильтров** — поиск по названию/бренду, фильтр по категории, фильтр по цене, сортировка
4. **Список товаров** — карточки товаров (изображение, название, бренд, цена, кнопка "В избранное", кнопка "В корзину")
5. **Кнопка "Загрузить ещё"** — пагинация
6. **Футер (Footer)** — контакты, ссылки на соцсети

**Динамический контент:** Список товаров загружается с `GET /api/items`

**Состояние загрузки:** Спиннер на весь экран

**Состояние ошибки:** Сообщение "Ошибка загрузки товаров"

---

### Страница 2: Аутентификация - Вход (`/auth/login`)
**Тип доступа:** Публичный

**Основные блоки:**
1. **Форма входа** — поля Email, Пароль, кнопка "Войти"
2. **Ссылка на регистрацию** — "Нет аккаунта? Зарегистрироваться"
3. **Ссылка на восстановление пароля**

**Динамический контент:** Отправка запроса на `POST /api/auth/login`

**Состояние ошибки:** Красное сообщение "Неверные учётные данные"

---

### Страница 3: Аутентификация - Регистрация (`/auth/register`)
**Тип доступа:** Публичный

**Основные блоки:**
1. **Форма регистрации** — поля Имя, Email, Пароль, кнопка "Зарегистрироваться"
2. **Ссылка на вход** — "Уже есть аккаунт? Войти"

**Динамический контент:** Отправка запроса на `POST /api/auth/register`

**Состояние ошибки:** Сообщение о некорректных данных

---

### Страница 4: Детальная страница товара (`/item?id=:id`)
**Тип доступа:** Публичный

**Основные блоки:**
1. **Шапка** — навигация
2. **Информация о товаре** — название, бренд, категория, цена, количество на складе, изображение
3. **Кнопка "Добавить в избранное"** — доступна только авторизованным
4. **Кнопка "Добавить в корзину"**
5. **Кнопка "Назад"** — возврат в каталог

**Динамический контент:** `GET /api/items/:id`

---

### Страница 5: Личный кабинет (`/profile`)
**Тип доступа:** Приватный

**Основные блоки:**
1. **Информация о пользователе** — имя, email
2. **Форма добавления товара** — название, бренд, категория, цена, количество, URL изображения
3. **Список товаров пользователя** — карточки с кнопками "Редактировать", "Удалить"

**Динамический контент:** `GET /api/items`, `POST /api/items`, `PUT /api/items/:id`, `DELETE /api/items/:id`

---

### Страница 6: Избранное (`/favorites`)
**Тип доступа:** Приватный

**Основные блоки:**
1. **Список избранных товаров** — карточки товаров
2. **Кнопка "Удалить из избранного"** — для каждого товара

**Динамический контент:** `GET /api/favorites`, `DELETE /api/favorites/:id`

---

### Страница 7: Корзина (`/cart`)
**Тип доступа:** Приватный

**Основные блоки:**
1. **Список товаров в корзине** — карточки с количеством и ценой
2. **Кнопки изменения количества** (+/-)
3. **Итоговая сумма**
4. **Кнопка "Оформить заказ"**

**Динамический контент:** `GET /api/cart`, `POST /api/cart/add`, `PUT /api/cart/update/:id`, `DELETE /api/cart/remove/:id`

---

### Страница 8: Оформление заказа (`/checkout`)
**Тип доступа:** Приватный

**Основные блоки:**
1. **Форма доставки** — имя, телефон, адрес, комментарий
2. **Сводка по заказу** — товары, количество, итоговая сумма
3. **Кнопка "Подтвердить заказ"**

**Динамический контент:** `POST /api/orders/create`

---

### Страница 9: Мои заказы (`/orders`)
**Тип доступа:** Приватный

**Основные блоки:**
1. **Список заказов** — номер заказа, дата, статус, сумма
2. **Детали заказа** — список товаров, данные доставки

**Динамический контент:** `GET /api/orders`

---

### Страница 10: Настройки профиля (`/settings`)
**Тип доступа:** Приватный

**Основные блоки:**
1. **Форма редактирования** — имя, email, смена пароля
2. **Кнопка "Сохранить изменения"**

**Динамический контент:** `GET /api/auth/me`, `PUT /api/auth/update`
